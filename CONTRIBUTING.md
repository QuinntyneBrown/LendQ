# Contributing to LendQ

Thank you for your interest in contributing to LendQ! This guide will help you get started.

## Getting Started

1. **Fork** the repository and clone your fork locally.
2. Follow the [Quick Start](README.md#quick-start) instructions to set up your development environment.
3. Create a new branch from `main` for your work.

## Repository Structure

- `backend/` — Flask API
- `frontend/` — React + TypeScript SPA
- `e2e/` — Playwright end-to-end tests
- `docs/` — Requirements, architecture, UI assets, and OpenAPI contract
- `ops/` — Local development infrastructure

## Development Workflow

1. Make your changes in a feature branch.
2. Write or update tests as appropriate.
3. Ensure all existing tests pass before submitting.
4. Keep commits focused and write clear commit messages.

## Pull Requests

- Open a pull request against the `main` branch.
- Provide a clear description of what the PR does and why.
- Link any related issues.
- Keep PRs focused — one feature or fix per PR.

## Code Style

- **Backend (Python):** Follow PEP 8 conventions.
- **Frontend (TypeScript/React):** Follow the existing project conventions and linting rules.

## Reporting Issues

- Use GitHub Issues to report bugs or request features.
- Include steps to reproduce for bug reports.
- Provide as much context as possible.
